Assignments

This course is heaviy focused on self-directed learning, where you have certain tasks to accomplish by specific due dates, but you may work faster and actually should strive to get ahead if you can. More material and resources can definitely be provided for any student(s) who wants to explore web tools more deeply. That said, the trick is that you will be working in groups of three so your pace will be somewhat constrained by how well your teammates and you support each other. Additionally, all of the groups will form a larger learning community where everyone helps each other with their individual learning goals.

Each task has a certain number of points associated with it. For some of the tasks, it's just a binary: did you do it or not in a timely manner? For others, there's some wiggle room for how many points you receive, and receiving the maximum means that not only have you done the task but you've done it such that I can tell there was thought and care put into it. For all of the group work, obviously, a lot will depend on the coordination of the group, but each student will be keeping a personal Learning Journal where they can indicate their thoughts on the work and how they could improve it. In fact, in many cases, good and detailed reflective writing in this journal is more important than just doing the task.

Reflective Learning Journal entries should be about one or two pages long and include:

  1. What you did. Include things that didn't work or alternative approaches you needed to come up with. Detail is important. Think of this as sort of a documentary or behind-the-scenes look at the process.
  2. What you learned. Include how you think you learned it and whether there was a particular resource or concept that you can highlight that helped you learn how to do the task.
  3. What worked well for the task. Include thoughts on group work and how the group coordinated and communicated and how this could be improved.
  4. If you think it helps, feel free to include screenshots of work.

  • Install the Discord mobile and/or desktop app, create an account (if you don't already have one), and join the group that will be sent out via Canvas.
  • Introduce yourself in Discord in the appropriate channel and also indicate what your schedule will be for the quarter so that we can form small groups better.

Register domain name and get hosting (2 pts):

  • Register a domain name (AKA a unique label that you'll own that you can have point to a specific address on the internet). I personally use namecheap, but there are a ton of options out there... It should cost less than $15 for a year.
  • Get hosting (AKA the address or location on the internet where you'll store your stuff). Freehosting seem like a good, easy-to-use choice and it's free, but, again, there are a ton out there. You could pay for hosting with namecheap or vice versa you could register (and pay) for a domain name with Freehosting, saving you a step.
  • Link your domain name to your hosting account by telling the domain name registrar where to lookup where your domain name is pointing to (AKA tell the domain name registrar what nameserver/domain name system (DNS) to use to store where your domain name points to). Each hosting company uses their own DNS. You can sort of think of these as like apartment buildings. When someone tries to go to your domain, your registrar will say, ok, now go to this DNS (apartment building) to look up where that domain actually resides (the specific apartment).
  • It'll look something like NS1.FREEHOSTING.COM.
  • Note: All of this takes time. Registering a domain name, getting a hosting account, linking DNS... they can take time for the internet to update, and, also, it can take time to verify who you are with the service providers (AKA they may send you an email confirmation, etc.), so be prepared for this initial set up to take a day or two.

Install WordPress and set up a portfolio (5 pts + 1 for helping someone else out with this):

  • WordPress comes in two flavors. There's the free version, which you may have used before for other courses, at wordpress.com, but there's also the software platform that you can install onto any web server. This latter version is what you'll be using, and it comes with much more control over set up and customization.
  • Pick theme, browse and install plugins, set up rapid portfolio.
  • Steps to follow and tips and tricks can be found in the WordPress document in our Google Team Drive.

Cascading Style Sheets (Groups)

Create a basic HTML + CSS page with another student’s GitHub Pages page (10 pts).

  • Same steps as above with the HTML page, basically. You can duplicate the first page on the first student's GitHub Pages and work from there, as I did with the Sample Pages. Refer to the Sample CSS Page, have all three students iterate off one another's work, and comment your code!
  • The page must include at least:
    • various ways to specify a style at different hierarchies: with an externally linked stylesheet text file using the <link rel=""> tag in the <head> tag; using an embedded stylesheet using the <style> tag in the <head> of the document; and in-line as a style="" attribute within another HTML tag.
    • various ways to specify what should actually be styled: a specific tag like, say, you want all <a> tags to be styled in a particular way; only certain subsets of tags by using the class="" attribute in a tag and specifying .class in the stylesheet; or specific unique elements that need to be styled a certain way through using the id="" attribute and #id in the stylesheet.
    • various style changes: font size, weight, and treatment; position and alignment of elements; manipulation of images; and hover and visibility changes links or other elements

Bootstrap CSS Framework (Groups)

Bootstrap is a set of predefined styles and functions (AKA a framework) that a group of developers have created and shared with the world as a resource for others so they don't have to invent the wheel with each new website they create. It includes common ways to structure and layout a page, using a grid system.

Create a Bootstrap version of the same page on the third student’s GitHub Pages page (10 pts).

  • As above, refer to the Sample Bootstrap Page, have all three students iterate off one another's work, and comment your code!
  • Include:
    • navbar, grids, flexboxes, and other effects

Javascript (Groups)

Ok, here's where it gets complicated fast. Javascript is a programming language and many consider the skill ceiling for it much higher than just HTML/CSS. You could study and use Javascript for years and still not feel like an expert. That means this course must focus on introducing you to the resources available for continued learning after the course is over. There's some fundamental components to any programming language that you should learn. Getting that foundation down will help with all programming languages in the future (and generally help you think logically in other disciplines as well!).

For this assignment, you'll follow the same process as the others in creating a page on a student's GitHub Pages (just rotate to the first student's account), but it will be difficult just seeing from the code whether all three students "get it." This is another reason why the Learning Journal is so important. Even if you don't fully understand, I want to see evidence that you're earnestly trying and that you have a plan for moving forward.

As above, refer to the Sample JS Page, work iteratively together, comment code.

Create a page that does the following (10 pts):

  • Declare a variable and set its value.
  • Use an array (AKA a table of variables stored as a set).
  • Create a function.
  • Throw in some conditional logic (AKA an if-then statement or a switch of some sort).
  • Create a loop that runs multiple times until some end condition is met.
  • Make some element execute the function when clicked on or is interacted with somehow by the user.

Each student should also add to their Learning Journal a reflection on this assignment (5 pts).
